# 公開手順（Chrome Web Store）

「Marker:HELPER」を Chrome Web Store に公開・更新するための手順をまとめる。本拡張はビルドステップを持たないバニラ JS/HTML/CSS のため、配布物は「出荷対象ファイルを ZIP 化するだけ」で作れる。

## 前提

- 公開には Google アカウントと **Chrome Web Store デベロッパー登録（$5・初回のみ／1アカウントで最大20拡張まで）** が必要。
- 登録は [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole) から行う。
- 受付は **Manifest V3 のみ**（本拡張は対応済み）。
- 審査は通常24時間以内のことが多いが、**広い権限（`<all_urls>`）があると詳細審査となり1〜4週間**かかることがある（後述）。

## 1. バージョンを確認・更新する

`manifest.json` の `version` を確認する。**再公開のたびに番号を上げる必要がある**（同じ番号では再アップロードできない）。

```jsonc
// manifest.json
"version": "1.0.0"
```

更新版を出すときは `1.0.1` / `1.1.0` などに上げてから ZIP を作る。

## 2. 配布用 ZIP を作る

出荷物に**不要なファイルを含めない**。ストアに含めるのは以下だけ。

| 含める | 含めない |
|---|---|
| `manifest.json` | `node_modules/`（テスト専用 dev 依存） |
| `background.js` | `test/` `e2e/` `test-results/`（テスト一式） |
| `content/` | `docs/`（本ドキュメント等） |
| `popup/` | `package.json` `package-lock.json` `*.config.*` |
| `panel/` | `.git/` `.gitignore` `CLAUDE.md` `TESTING.md` |
| `shared/` | `icons/icon.svg`（元データ。PNG のみ出荷） |
| `icons/*.png` | |

リポジトリのルートで次を実行する。

```bash
zip -r marker-helper-$(node -p "require('./manifest.json').version").zip \
  manifest.json background.js content popup panel shared icons \
  -x "icons/icon.svg"
```

> 解凍したら直下に `manifest.json` が来る形にすること（余計な親フォルダを入れない）。

## 3. アイテムを登録し ZIP をアップロードする

1. Developer Dashboard で「**新しいアイテム**」を選択。
2. 手順 2 の ZIP をアップロード。
3. 既存アイテムの更新時は、対象アイテムを開いて「パッケージ」→ 新しい ZIP をアップロードする。

## 4. ストア掲載情報を入力する

- **アイコン 128×128**: `icons/icon128.png`（用意済み）。
- **スクリーンショット**: **1280×800 または 640×400** を最低 1 枚・最大 5 枚。**実際の拡張UI**を写すこと（マーケ画像不可）。
  - 拡張を実際に動かし（`chrome://extensions` → デベロッパーモード → 「パッケージ化されていない拡張機能を読み込む」）、マーキング・サイドパネル・撮影などの画面を撮る。
- **説明文 / カテゴリ / 言語**: 機能（ホバー強調、番号付きマーク、CSS/XPath セレクタ確認・コピー、撮影、レポート出力など）を記載。

## 5. プライバシーを申告する（必須要件あり）

本拡張は**データを外部送信しない**。永続化はすべて端末ローカルの `chrome.storage`（`local` / `session`）に閉じている。

- 「個人情報・ユーザーデータを収集しない」旨を申告する。
- リモートコード（外部 JS の読み込み・eval 等）は使用していない旨も申告できる。
- ダッシュボードの**データ開示 / 限定使用の証明**欄を記入する（更新・公開のたびに必要）。

> **【必須】プライバシーポリシー URL**: 本拡張は `activeTab` / `storage` / ホスト権限（`<all_urls>`）を使うため、**公開アクセス可能なプライバシーポリシー URL の登録が必須**。雛形を [`docs/PRIVACY.md`](./PRIVACY.md) に用意済み。これを GitHub Pages 等で公開し、その URL を「プライバシー」→「プライバシーポリシーの URL」に登録する。

## 6. 権限の正当化を記載する（審査の鍵）

`<all_urls>` のような広いホスト権限は**詳細審査（in-depth review）**を誘発し、審査が長引く。却下の典型は「権限と機能の不一致」「使っていない権限」なので、各権限の用途を明確に書く。

| 権限 | 用途説明 |
|---|---|
| `host_permissions: ["<all_urls>"]` | ユーザーが閲覧中の任意のページ上の要素にマーカーを描画・撮影するため、全 URL へのアクセスが必要。別ウィンドウ表示時の `captureVisibleTab` にも必要。ページ内容の外部送信には使用しない。 |
| `activeTab` | アクティブタブの content script へのメッセージ送信とビューポート撮影に使用。 |
| `tabs` | アクティブタブの特定とメッセージ送信に使用。 |
| `storage` | 設定（スタイル・ラベル表示・マイカラー）と自動保存マークの端末ローカル保存に使用。外部送信なし。 |
| `sidePanel` | マーク一覧のサイドパネル表示に使用。 |

`commands`（キーボードショートカット）は利便性向上のためで、説明不要なことが多い。

## 7. 公開範囲を選び申請する

- **公開（Public）** / **限定公開（Unlisted・一覧非表示だがURLを知る人はインストール可）** / **非公開（Private・指定ユーザーのみ、テスト用）** から選択。
- 「審査のために送信」で申請完了。

## 審査期間の目安（2026年時点）

- **通常審査**: 多くは24時間以内。
- **詳細審査**（`<all_urls>` 等の広い権限・コード量が多い・新規アカウント）: **1〜2週間、長いと3〜4週間**。
- 本拡張は `<all_urls>` を使うため、詳細審査になる前提でスケジュールを見ておく。

## 更新時のチェックリスト

- [ ] `manifest.json` の `version` を上げた
- [ ] `npm test` がグリーン（出荷物に影響する回帰がない）
- [ ] 不要ファイルを除いた ZIP を再作成した
- [ ] 機能を追加した場合、権限の追加有無と説明を見直した
- [ ] プライバシーポリシー（`docs/PRIVACY.md`）の内容が最新の挙動と一致している
- [ ] スクリーンショット・説明文を最新の挙動に合わせた

# ストアを使わない配布方法

Chrome Web Store を使わずに配布する方法もあるが、**Chrome の仕様により一般ユーザー向けには大きな制約がある**。用途別に整理する。

## 1. デベロッパーモードで読み込む（unpacked）

最も手軽。配布物のフォルダをそのまま渡し、各自に読み込んでもらう。

- 手順: `chrome://extensions` → 「デベロッパー モード」ON → 「パッケージ化されていない拡張機能を読み込む」→ フォルダを選択
- **向き**: 開発・自分用・少人数の技術者チーム
- **欠点**: 起動のたびに警告が出る／自動更新されない／配布フォルダを各自が保持する必要がある

本プロジェクトは普段この方法で動作確認しているため、そのまま配れる。

## 2. `.crx` ファイルにして配る（実質不可）

`chrome://extensions` の「拡張機能をパッケージ化」で `.crx` を生成できるが、現在の Chrome は**ストア外からの `.crx` 直接インストールを原則ブロック**する。一般ユーザーへの直接配布には使えない。

## 3. 自己ホスティング / エンタープライズ配布（組織向け）

- **自己ホスティング**（自前サーバーに `.crx` と `updates.xml` を置く）は、**管理環境（エンタープライズポリシー）でのみ**機能する。Windows/macOS の一般ユーザーは自己ホスト拡張をインストールできない。
- **エンタープライズポリシー**（`ExtensionInstallForcelist` / `ExtensionSettings` 等）で、管理下端末へ強制インストール・許可/拒否リスト適用が可能。
- **【新】組織向け公開（organization publishing・2026年2月導入）**: 承認した外部組織へ専用リンクで非公開配布できる。強制インストールやコレクション整理に対応。

## 4. 限定公開（Unlisted）でストアを使う

「検索に出したくない／広く公開したくない」だけなら、これが現実的。

- ストアには登録するが、**検索結果に出ず、URL を知っている人だけがインストール可能**
- `.crx` 直配布の制約を回避でき、自動更新も効く
- 審査・$5 登録は必要

## 配布対象別の選び方

| 状況 | 推奨方法 |
|---|---|
| 自分・技術者数名で使う | **1. unpacked で配布**（無料・即時） |
| 会社/組織の端末に配る | **3. エンタープライズ配布 / 組織向け公開** |
| 一般の人に配りたいが検索には出したくない | **4. 限定公開（Unlisted）** |
| 広く一般公開したい | 通常のストア公開（本書の前半） |

> **注意**: 一般ユーザーに「ストアを完全に使わず」配る確実な方法は実質ない（Chrome がセキュリティ上ブロックするため）。広く配るなら最低でも「限定公開」でストアの仕組みに乗るのが現実解。
